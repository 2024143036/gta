# MobileBERT를 활용한 GTA 시리즈 리뷰 감성 분석 및 이슈 분석 프로젝트

---

## 1. 개 요

**"역대 최고 흥행작 GTA 시리즈, 글로벌 유저들은 어떤 요소에 열광하고 분노했는가?"**

게임 산업은 현대 대중문화 및 디지털 엔터테인먼트 마켓에서 가장 영향력 있고 거대한 규모를 자랑하는 콘텐츠 영역이다. 그중에서도 록스타 게임즈(Rockstar Games)의 **GTA(Grand Theft Auto)** 시리즈는 전 세계적으로 수억 장의 판매고를 기록하며 오픈월드 장르의 교과서이자 상징으로 자리 잡았다. 글로벌 게임 유통 플랫폼인 스팀(Steam)에 누적되는 대규모의 실시간 유저 리뷰는 게임의 지속적인 장기 흥행 여부와 유저 만족도를 직관적으로 보여주는 핵심 지표이다.

본 프로젝트에서는 GTA 시리즈의 기념비적인 흥행작인 **GTA 5**와 독보적인 디테일과 스토리 라인으로 명작이라 평가받는 **GTA 4**를 대상으로 대규모 리뷰 데이터셋을 분석하고자 한다. 가공되지 않은 60,000건의 글로벌 유저 영문 리뷰 데이터를 직접 수집 및 전처리한 후, 복잡한 문맥적 특성을 반영한 3클래스(부정, 중립, 긍정) 감성 지표를 구축했다.

이를 구글의 초경량 고성능 언어 모델인 **MobileBERT**에 전이 학습(Fine-tuning)하여 로컬 환경에서도 감성 분류가 가능한 분석 파이프라인을 구현하고, 자동 라벨링과 수동 라벨링의 차이가 모델 성능에 어떤 영향을 주는지 비교하는 것을 목적으로 한다.

본 프로젝트에서 확인하고자 하는 핵심 질문은 다음과 같다.

1. GTA4와 GTA5 유저 리뷰는 어떤 감성 분포를 보이는가?
2. 컴퓨터가 규칙 기반으로 라벨링한 결과와 사람이 직접 라벨링한 결과는 어떻게 다른가?
3. 같은 3,000개의 리뷰를 사용했을 때 자동 라벨 데이터와 수동 라벨 데이터 중 어떤 데이터가 MobileBERT 학습에 더 적합한가?
4. 목표 검증 정확도 0.85에 도달하지 못한 경우, 그 원인은 데이터 분포와 라벨링 품질 측면에서 어떻게 해석할 수 있는가?

|구분|게임명|수집 기준|수집 개수|

## 3. 학습 데이터 구축

**총 데이터가 5만건이라고 한다면 이 중에서 10% 정도 혹은 2,000 ~ 3,000건 정도의 데이터를 학습데이터로 만든다. 학습 데이터를 추출할 때 단순한 랜덤이 아닌 데이터의 분포를 보고 판단한다. 예를 들어 부정데이터(특정 라벨의 데이터)의 비중이 매우 작을 때는 부정과 긍정을 1:1로 구축할 수 있다. 혹은 일자별(특정한 기준-attribute)로 비율을 바탕으로 학습데이터를 구축할 수 있다.**

본 프로젝트에서는 전체 리뷰 60,000개 중 3,000개를 추출하여 라벨링 데이터로 사용하였다. GTA4와 GTA5에서 각각 1,500개씩 추출하였기 때문에 특정 게임에 데이터가 치우치지 않도록 구성하였다.

### 3.1 데이터 라벨링

본 프로젝트에서는 같은 3,000개의 리뷰에 대해 두 가지 방식으로 라벨링을 진행하였다.

1. 컴퓨터 자동 라벨링
2. 사람 수동 라벨링

자동 라벨링은 규칙 기반으로 진행하였다. 리뷰 텍스트에 특정 단어가 포함되어 있거나 Steam의 추천 여부를 기준으로 긍정, 중립, 부정 라벨을 부여하였다. 처음에는 관련없음(3) 라벨도 사용하였으나, 관련없음 라벨의 수가 매우 적어 최종 학습에서는 부정(0)으로 통합하였다.

수동 라벨링은 자동 라벨링과 같은 3,000개의 리뷰를 사람이 직접 확인하고 0, 1, 2 라벨을 부여하는 방식으로 진행하였다. 같은 리뷰를 대상으로 두 가지 라벨링 방식을 적용했기 때문에, 자동 라벨과 수동 라벨의 차이를 직접 비교할 수 있다.

라벨 기준은 다음과 같다.

|라벨|의미|설명|
|---:|---|---|
|0|부정|게임에 대한 불만, 오류, 최적화 문제, 재미없음, 관련없는 리뷰 등|
|1|중립|장점과 단점이 섞여 있거나 명확히 긍정/부정으로 판단하기 어려운 리뷰|

### 3.2 학습 데이터

MobileBERT 학습에서는 자동 라벨 데이터와 수동 라벨 데이터를 각각 사용하였다. 두 데이터는 같은 3,000개의 리뷰를 사용하지만 라벨링 방식이 다르다.

|모델|학습 데이터|설명|
|---|---|---|
|모델 A|자동 라벨 데이터|컴퓨터가 규칙 기반으로 라벨링한 데이터|
|모델 B|수동 라벨 데이터|사람이 직접 라벨링한 데이터|

자동 라벨 데이터는 라벨 분포가 불균형했기 때문에, 학습 시 라벨별 개수를 맞추기 위해 균등 샘플링을 사용하였다. 자동 라벨 데이터의 경우 가장 적은 라벨은 부정(0) 라벨 252개였고, 이에 맞추어 라벨별 학습 202개, 검증 50개를 사용하였다.

|구분|라벨별 개수|총 개수|
|---|---:|---:|
|학습 데이터|202개|606개|
|검증 데이터|50개|150개|

수동 라벨 데이터도 같은 방식으로 균등 샘플링하였다. 수동 라벨 데이터의 경우 가장 적은 라벨은 중립(1) 라벨 493개였고, 이에 맞추어 라벨별 학습 395개, 검증 98개를 사용하였다.

|구분|라벨별 개수|총 개수|
|---|---:|---:|
|학습 데이터|395개|1185개|
|검증 데이터|98개|294개|

## 4. MobileBERT 모델 학습 (Fine-tuning)

본 프로젝트에서는 사전학습 언어모델인 **MobileBERT**를 사용하여 GTA4·GTA5 유저 리뷰 감성 분류 모델을 학습하였다. 자동 라벨 데이터와 수동 라벨 데이터를 각각 동일한 구조로 학습하여 결과를 비교한다.

### 4.1 학습 설정

|항목|설정|
|---|---|
|모델|`google/mobilebert-uncased`|
|분류 클래스 수|3개|
|라벨|0: 부정, 1: 중립, 2: 긍정|
|학습/검증 비율|약 8:2|
|Batch size|16|
|Epoch|4|
|Optimizer|AdamW|
|개발 환경|Python, PyCharm, Anaconda|
|실행 장치|CPU|

### 4.2 자동 라벨 데이터 학습 결과

자동 라벨 데이터를 사용하여 MobileBERT를 학습한 결과는 다음과 같다.

|Epoch|학습 오차|학습 정확도|검증 정확도|
|---:|---:|---:|---:|
|1|555083.2045|0.2954|0.3067|
|2|2.6012|0.6815|0.6600|
|3|0.8789|0.7442|0.7200|
|4|0.6955|0.7706|0.7533|

자동 라벨 데이터의 최종 검증 정확도는 다음과 같다.

|학습 데이터|최종 학습 정확도|최종 검증 정확도|
|---|---:|---:|
|자동 라벨 데이터|0.7706|0.7533|

자동 라벨 데이터 학습에서는 최종 검증 정확도 0.7533을 기록하였다. 이는 초기 4분류 실험보다 향상된 결과이지만, 목표 정확도인 0.85에는 도달하지 못하였다. 자동 라벨 데이터에서 부정 라벨의 수가 252개로 적어 균등 샘플링 과정에서 전체 학습 데이터 수가 제한되었고, 이 점이 모델 성능에 영향을 준 것으로 볼 수 있다.

### 4.3 수동 라벨 데이터 학습 결과

수동 라벨 데이터를 사용하여 MobileBERT를 학습한 결과는 다음과 같다.

|Epoch|학습 오차|학습 정확도|검증 정확도|
|---:|---:|---:|---:|
|1|972896.8665|0.4717|0.4422|
|2|0.8861|0.7544|0.6667|
|3|0.6722|0.8169|0.7177|
|4|0.5713|0.8363|0.7381|

수동 라벨 데이터의 최종 검증 정확도는 다음과 같다.

|학습 데이터|최종 학습 정확도|최종 검증 정확도|
|---|---:|---:|
|수동 라벨 데이터|0.8363|0.7381|

수동 라벨 데이터는 자동 라벨 데이터보다 학습 데이터 수가 많았지만, 최종 검증 정확도는 0.7381로 자동 라벨 데이터의 0.7533보다 약간 낮게 나타났다. 이는 수동 라벨링이 리뷰 문맥을 더 세밀하게 반영하는 대신, 중립과 부정의 경계가 애매한 리뷰가 포함되어 모델이 분류하기 어려웠기 때문으로 해석할 수 있다.

## 5. 문제제기에 대한 결과

본 프로젝트의 핵심 문제는 **GTA4와 GTA5의 같은 리뷰 3,000개를 사용했을 때, 컴퓨터 자동 라벨링과 사람 수동 라벨링 중 어떤 방식이 MobileBERT 학습에 더 적합한가**이다.

자동 라벨 데이터로 학습한 결과, 최종 검증 정확도는 0.7533으로 나타났다. 관련없음(3) 라벨을 부정(0) 라벨로 통합하여 3개 클래스로 학습한 결과, 이전보다 성능은 향상되었지만 목표 정확도 0.85에는 도달하지 못하였다.

수동 라벨 데이터로 학습한 결과, 최종 검증 정확도는 0.7381로 나타났다. 수동 라벨 데이터는 자동 라벨 데이터보다 학습 데이터 수가 더 많았지만, 최종 검증 정확도는 자동 라벨보다 낮았다. 이는 수동 라벨이 문맥을 더 반영하는 과정에서 중립과 부정의 경계가 더 복잡하게 나타났고, 모델이 해당 차이를 학습하기 어려웠기 때문으로 볼 수 있다.

최종 비교 표는 다음과 같다.

|구분|최종 학습 정확도|최종 검증 정확도|해석|
|---|---:|---:|---|
|자동 라벨|0.7706|0.7533|수동 라벨보다 검증 정확도가 약간 높음|
|수동 라벨|0.8363|0.7381|학습 정확도는 높지만 검증 정확도는 자동 라벨보다 낮음|

자동 라벨과 수동 라벨 모두 목표 정확도 0.85에는 도달하지 못하였다. 그러나 두 모델 모두 epoch가 증가할수록 학습 정확도와 검증 정확도가 상승하는 흐름을 보였기 때문에, MobileBERT가 GTA 리뷰의 감성 패턴을 어느 정도 학습하고 있음을 확인할 수 있었다.

두 라벨링 방식의 비교 결과, 본 실험에서는 자동 라벨 데이터가 최종 검증 정확도 기준으로 더 높은 성능을 보였다. 하지만 차이가 크지 않으며, 전체적으로는 라벨 불균형과 중립 라벨의 애매함이 성능 향상의 가장 큰 한계로 작용하였다.

## 6. 느낀점 및 개선방향

이번 프로젝트를 통해 Steam 유저 리뷰 데이터를 직접 수집하고, 같은 데이터에 대해 자동 라벨링과 수동 라벨링을 적용하여 비교할 수 있었다. 단순히 데이터를 많이 수집하는 것보다 라벨의 품질과 분포가 모델 성능에 큰 영향을 준다는 점을 확인할 수 있었다.

자동 라벨링은 짧은 시간에 많은 데이터를 처리할 수 있다는 장점이 있었다. 그러나 단순 규칙이나 추천 여부만을 기준으로 라벨을 부여하기 때문에, 리뷰 문장의 문맥을 정확하게 반영하기 어렵다는 한계가 있었다. 특히 게임 리뷰는 긍정과 부정이 한 문장 안에 함께 나타나는 경우가 많아 중립과 부정의 구분이 쉽지 않았다.

수동 라벨링은 사람이 직접 문맥을 확인할 수 있다는 장점이 있었지만, 중립과 부정처럼 경계가 애매한 라벨에서는 일관성을 유지하기 어려웠다. 또한 라벨 분포가 불균형할 경우, 가장 적은 라벨의 개수에 맞추어 균등 샘플링을 해야 하므로 실제 학습에 사용되는 데이터 수가 줄어드는 문제가 발생하였다.

향후 개선 방향은 다음과 같다.

1. 부정 및 중립 리뷰 데이터를 더 많이 확보하여 라벨 불균형을 줄인다.
2. 자동 라벨링 규칙을 개선하여 단순 추천 여부뿐만 아니라 리뷰 문맥을 더 반영한다.
3. 수동 라벨링 데이터의 기준을 명확히 하여 라벨 품질을 높인다.
4. CPU가 아닌 GPU 환경에서 학습하여 더 많은 epoch와 다양한 하이퍼파라미터 실험을 진행한다.
5. 감성 분류 목표를 더 단순화하여 부정/긍정 2분류 실험도 추가로 진행한다.
6. 최종적으로 자동 라벨과 수동 라벨의 차이가 크게 나타나는 리뷰를 분석하여 라벨링 기준을 개선한다.
